# MongoDB - Consultas avanzadas


# Recordatorio: estructura de MongoDB

La información se organiza de la siguiente manera:

```
Servidor
│
├── Base de datos
│      │
│      ├── Colección
│      │      │
│      │      ├── Documento
│      │      ├── Documento
│      │      └── Documento
```

Cada documento está almacenado en formato **BSON**, similar a JSON, y puede contener:

* Campos simples
* Objetos anidados
* Arrays

Ejemplo:

```javascript
{
    titulo: "Introducción a MongoDB",
    activo: true,
    categoria: "tecnología",

    autor: {
        nombre: "Juan",
        edad: 30
    },

    comentarios: [
        {
            autor: "Ana",
            likes: 12
        },
        {
            autor: "Pedro",
            likes: 3
        }
    ]
}
```

# Operadores avanzados

## `$elemMatch`

Permite buscar documentos donde **al menos un elemento de un array** cumple determinadas condiciones.

Ejemplo:

```javascript
db.articulos.find({
    comentarios: {
        $elemMatch: {
            autor: "Ana",
            likes: {
                $gt: 10
            }
        }
    }
})
```

Se lee como:

> Buscar artículos donde exista un comentario cuyo autor sea Ana y tenga más de 10 likes.

---

## `$regex`

Permite buscar texto utilizando expresiones regulares.

Ejemplo:

```javascript
db.articulos.find({
    titulo: {
        $regex: /mongodb/i
    }
})
```

La bandera **`i`** significa:

> Ignorar mayúsculas y minúsculas.

También podría escribirse:

```javascript
db.articulos.find({
    titulo: {
        $regex: "mongodb",
        $options: "i"
    }
})
```

---

## `$exists`

Permite verificar si un campo existe.

Buscar documentos que tengan resumen:

```javascript
db.articulos.find({
    resumen: {
        $exists: true
    }
})
```

Buscar documentos que NO tengan resumen:

```javascript
db.articulos.find({
    resumen: {
        $exists: false
    }
})
```

---

# Operadores lógicos

MongoDB permite combinar múltiples condiciones.

## `$and`

Todas las condiciones deben cumplirse.

```javascript
db.articulos.find({
    $and: [
        { activo: true },
        { categoria: "tecnología" }
    ]
})
```

---

## `$or`

Al menos una condición debe cumplirse.

```javascript
db.articulos.find({
    $or: [
        { categoria: "tecnología" },
        { categoria: "educación" }
    ]
})
```

---

## `$not`

Niega una condición.

```javascript
db.articulos.find({
    activo: {
        $not: {
            $eq: true
        }
    }
})
```

También suele ser más sencillo escribir:

```javascript
db.articulos.find({
    activo: false
})
```

---

## `$nor`

Ninguna condición debe cumplirse.

```javascript
db.articulos.find({
    $nor: [
        { categoria: "tecnología" },
        { categoria: "educación" }
    ]
})
```

---

# Combinando operadores

Buscar artículos:

* activos
* cuya categoría sea tecnología o educación

```javascript
db.articulos.find({
    $and: [
        {
            activo: true
        },
        {
            $or: [
                { categoria: "tecnología" },
                { categoria: "educación" }
            ]
        }
    ]
})
```

También podría escribirse de forma más simple:

```javascript
db.articulos.find({
    activo: true,
    $or: [
        { categoria: "tecnología" },
        { categoria: "educación" }
    ]
})
```

MongoDB interpreta que ambas condiciones deben cumplirse.

# ¿Por qué son importantes estos operadores?

En aplicaciones reales los documentos suelen contener estructuras complejas.

Ejemplo:

```javascript
{
    titulo: "...",
    autor: {
        nombre: "Juan"
    },
    etiquetas: [
        "mongodb",
        "backend",
        "node"
    ],
    comentarios: [
        {
            usuario: "Ana",
            likes: 12
        },
        {
            usuario: "Pedro",
            likes: 5
        }
    ]
}
```

Gracias a estos operadores podemos consultar solamente la información que necesitamos sin traer todos los documentos y procesarlos desde Node.js.

Esto mejora:

* el rendimiento
* el consumo de memoria
* el tiempo de respuesta de la aplicación

---

# Buenas prácticas

* Utilizar filtros lo más específicos posible.
* Evitar traer documentos innecesarios.
* Usar proyecciones (`find({}, { campo: 1 })`) cuando solo se necesiten algunos campos.
* Crear índices sobre los campos más consultados.
* Evitar abusar de `$regex`, especialmente cuando el patrón comienza con `.*`, ya que puede impedir el uso de índices y volver las consultas más lentas.

# Resumen

| Operador     | Función                                |
| ------------ | -------------------------------------- |
| `$elemMatch` | Buscar elementos dentro de un array.   |
| `$regex`     | Buscar texto mediante patrones.        |
| `$exists`    | Verificar si un campo existe.          |
| `$and`       | Todas las condiciones deben cumplirse. |
| `$or`        | Al menos una condición debe cumplirse. |
| `$not`       | Niega una condición.                   |
| `$nor`       | Ninguna condición debe cumplirse.      |

## operadores de comparación
| Operador | Significado                                |
| -------- | ------------------------------------------ |
| `$gt`    | Mayor que (`>`)                            |
| `$gte`   | Mayor o igual que (`>=`)                   |
| `$lt`    | Menor que (`<`)                            |
| `$lte`   | Menor o igual que (`<=`)                   |
| `$eq`    | Igual a (`=`)                              |
| `$ne`    | Distinto de (`!=`)                         |
| `$in`    | El valor debe estar dentro de una lista    |
| `$nin`   | El valor no debe estar dentro de una lista |
