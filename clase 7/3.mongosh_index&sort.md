## Índices (Indexes)

Un **índice** es una estructura de datos que MongoDB crea para acelerar las búsquedas sobre uno o más campos.

Sin un índice, MongoDB debe recorrer todos los documentos de la colección hasta encontrar los que coinciden con la consulta. Esto se conoce como **Collection Scan**.

Con un índice, MongoDB consulta primero el índice y luego accede directamente a los documentos encontrados, reduciendo considerablemente el tiempo de búsqueda.

### Crear un índice

```javascript
db.usuarios.createIndex({
    edad: 1
})
```

El valor `1` indica que el índice se almacena en **orden ascendente**.

También puede utilizarse:

```javascript
db.usuarios.createIndex({
    edad: -1
})
```

que crea el índice en **orden descendente**.

### ¿Para qué sirve el `1` o el `-1`?

* `1` → Ascendente (menor a mayor, A-Z)
* `-1` → Descendente (mayor a menor, Z-A)

Para búsquedas simples (`find({ edad: 20 })`) ambos funcionan prácticamente igual.

La dirección del índice cobra importancia principalmente cuando se ordenan resultados mediante `sort()` o cuando se crean índices compuestos.

### ¿Cuándo conviene crear un índice?

Es recomendable crear índices sobre campos que:

* Se consultan frecuentemente.
* Se utilizan para ordenar resultados.
* Contienen muchos valores diferentes (por ejemplo: email, DNI o username).

No es recomendable crear índices sobre todos los campos, ya que cada índice:

* Ocupa espacio en disco.
* Consume memoria RAM.
* Debe actualizarse cada vez que se inserta, modifica o elimina un documento.

Demasiados índices pueden ralentizar las operaciones de escritura.

> **Nota:** El campo `_id` posee un índice creado automáticamente por MongoDB.

# Ordenar resultados (`sort()`)

El método `sort()` permite ordenar los documentos devueltos por una consulta.

## Orden ascendente

```javascript
db.usuarios.find().sort({
    edad: 1
})
```

Resultado:

```text
18
20
25
30
40
```

---

## Orden descendente

```javascript
db.usuarios.find().sort({
    edad: -1
})
```

Resultado:

```text
40
30
25
20
18
```

---

## Ordenar por texto

Por nombre (A-Z):

```javascript
db.usuarios.find().sort({
    nombre: 1
})
```

Por nombre (Z-A):

```javascript
db.usuarios.find().sort({
    nombre: -1
})
```

---

## Ordenar por varios campos

También es posible ordenar utilizando más de un campo.

Por ejemplo, ordenar primero por ciudad y luego por edad:

```javascript
db.usuarios.find().sort({
    ciudad: 1,
    edad: -1
})
```

MongoDB ordenará:

1. Primero por ciudad (A-Z).
2. Dentro de cada ciudad, por edad (de mayor a menor).

---

## Resumen

| Comando                      | Descripción                               |
| ---------------------------- | ----------------------------------------- |
| `createIndex({ campo: 1 })`  | Crea un índice ascendente.                |
| `createIndex({ campo: -1 })` | Crea un índice descendente.               |
| `sort({ campo: 1 })`         | Ordena resultados de menor a mayor o A-Z. |
| `sort({ campo: -1 })`        | Ordena resultados de mayor a menor o Z-A. |

> **Importante:** Un índice **no cambia el orden en que se almacenan los documentos** en la colección. Simplemente crea una estructura adicional que permite encontrarlos y ordenarlos de forma más eficiente.
