# MongoDB - Paginación y optimización

# Paginación

## Limitar resultados

```javascript
db.productos.find().limit(10)
```

---

## Omitir resultados

```javascript
db.productos.find().skip(20)
```

---

## Página 1 (10 por página)

```javascript
db.productos.find().skip(0).limit(10)
```

---

## Página 2 (10 por página)

```javascript
db.productos.find().skip(10).limit(10)
```

---

## Página 3 (10 por página)

```javascript
db.productos.find().skip(20).limit(10)
```

---

## Fórmula

```text
skip = (pagina - 1) * registrosPorPagina
```

---

# Paginación por cursor

Primeros 10 documentos:

```javascript
db.productos.find().limit(10)
```

Siguientes 10 (a partir del último `_id` obtenido):

```javascript
const lastId = ObjectId("507f1f77bcf86cd799439011")

db.productos.find({
    _id: {
        $gt: lastId
    }
}).limit(10)
```

# Índices

## Crear índice simple

```javascript
db.usuarios.createIndex({
    email: 1
})
```

---

## Crear índice compuesto

```javascript
db.usuarios.createIndex({
    ciudad: 1,
    edad: -1
})
```

---

## Ver índices

```javascript
db.usuarios.getIndexes()
```

---

## Eliminar índice

```javascript
db.usuarios.dropIndex({
    email: 1
})
```

---

# Analizar consultas

## Ver plan de ejecución

```javascript
db.usuarios.find({
    ciudad: "Madrid"
}).explain("executionStats")
```

---

## Modos de `explain()`

```javascript
.explain()
```

```javascript
.explain("queryPlanner")
```

```javascript
.explain("executionStats")
```

```javascript
.explain("allPlansExecution")
```

---

# Qué mirar en `executionStats`

- `executionSuccess`
- `executionTimeMillis`
- `totalDocsExamined`
- `totalKeysExamined`

Idealmente:

- Pocos documentos examinados.
- Uso de índices.
- Bajo tiempo de ejecución.

# Buenas prácticas

- Crear índices en campos muy consultados.
- Crear índices para campos usados en `sort()`.
- Evitar demasiados índices.
- Usar `limit()` cuando no se necesiten todos los documentos.
- Preferir paginación por cursor en colecciones grandes.
- Utilizar `explain()` para verificar si la consulta usa índices.

---

# ¿Cuándo usar cada tipo de paginación?

| Técnica | Uso recomendado |
|----------|-----------------|
| `skip()` + `limit()` | Colecciones pequeñas o medianas. |
| Cursor (`_id`, fecha, etc.) | Colecciones grandes o APIs con muchos registros. |
